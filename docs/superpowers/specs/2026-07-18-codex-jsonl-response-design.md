# Codex JSONL 最终响应提取设计

## 目标

让 TMS 的 Codex CLI 提供商能够从 `codex exec --json` 的 JSONL 事件流中取得最终代理消息，并将其作为翻译结果 JSON 继续校验。

## 范围

- 在 Codex CLI 提供商内部增加 JSONL 最终消息提取函数。
- 只接受 `type` 为 `item.completed` 且 `item.type` 为 `agent_message` 的事件。
- 使用最后一个符合条件的 `item.text` 作为完成文本。
- 增加多事件 JSONL 的回归测试。

## 非目标

- 不改变 `codex exec --json` 调用、可执行文件配置或鉴权环境继承。
- 不解析其他 Codex 事件类型，也不保存原始事件流。
- 不修改 TMS 任务状态、审批、快照发布或前端。

## 设计

提供商收到子进程 stdout 后按换行拆分，忽略空行并逐条 JSON 解析。仅记录完成的代理消息事件；遍历结束后返回最后一条消息的文本。若某行不是有效 JSON，或不存在符合条件且具有字符串文本的代理消息，则抛出明确错误，不把不完整结果交给后续 schema 校验。

测试通过注入的命令执行器返回包含进度事件和两条代理消息的 JSONL，断言完成文本为最后一条代理消息的文本；另断言无最终代理消息时被拒绝。测试不调用真实 Codex CLI。

## 验证

1. 先新增 JSONL 场景的失败测试，并确认现有实现将完整 JSONL 作为文本返回。
2. 实现提取函数后运行同一测试，确认最终消息被正确返回。
3. 运行 `@poker/agents` 的测试、类型检查、lint 与构建。
4. 重启本地 TMS API，在浏览器创建 `codex-cli` 翻译任务，确认任务进入审批状态并展示条目。
