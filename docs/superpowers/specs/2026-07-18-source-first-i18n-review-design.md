# 中文源文优先的多语言审核台设计

## 目标

以 `zh-CN.json` 为中文源文与编号的唯一来源，AI 根据中文生成英文建议；用户在中文 Ant Design 审核表格中编辑并确认后，增量更新 `zh-CN.json` 与 `en.json`。

## 编号迁移

所有既有消息编号统一为六位数字格式 `P000001` 至 `P999999`。现有 `P00042` 等编号迁移为 `P000042`，并同步替换全仓库 `t()` 调用、测试、字典键和类型校验。新增条目从中文字典最大编号递增分配；到达 `P999999` 时拒绝新增。

## i18n 模型

删除 `catalog.json`。`zh-CN.json` 是全部编号集合与中文源文；`en.json` 必须具有完全相同的编号集合。加载及保存时，从英中文本直接提取 `{0}`、`{1}` 等占位符并逐条比较；不一致时拒绝写入。`t()` 保持按模板替换参数的运行时行为。

## 审核台

使用真实 Ant Design。页面使用中文文案和表格布局，列为“编号｜英文原文（可编辑）｜中文译文（可编辑）｜审核状态”。占位符不作为表格列展示，仅在编辑和确认写入时作为底层校验。用户可新增中文条目；AI 只生成英文建议，不覆盖人工中文源文。

## 写入流程

确认写入时，服务读取当前两份 locale 文件，合并已批准的增量改动，校验键集合和全部占位符。服务在同目录写入并 fsync 两个临时文件，再替换目标文件；若第二次替换失败，尝试恢复第一份原文件并返回失败。写入成功后任务记录为已写入。删除快照发布、候选生成和 `TMS_DATA_DIR` 生产发布链路。

## 范围与约束

- 复用用户已安装的 `antd` 依赖；不修改或提交 `pnpm-lock.yaml`；不 push、部署、合并或重写 Git 历史。
- 不保留 `catalog.json` 的兼容读取或候选文件命令。
- 测试仅使用临时目录副本，不改写真实 locale 源文件。
- 迁移前在测试中覆盖完整编号引用替换；迁移后全仓库类型检查与相关测试必须通过。

## 验证

1. 验证所有字典键与引用均为六位编号，且不存在 `catalog.json` 依赖。
2. 验证中文源文新增条目可自动分配编号，并由 AI 生成英文建议。
3. 验证表格可编辑英中文、占位符不匹配时拒绝确认、确认后两份 locale 文件增量更新。
4. 验证 Ant Design 中文审核台在本地浏览器完成新增、审核和写入流程。
