# Web3 Testnet Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a separately built Base Sepolia mode with wallet login, test-token custody, confirmed deposits, off-chain play, and voucher withdrawals.

**Architecture:** Reown AppKit is isolated behind the Web3 client entry. Foundry contracts hold test tokens; a durable indexer credits the double-entry escrow ledger; an operator signer issues EIP-712 withdrawal vouchers.

**Tech Stack:** Solidity, Foundry, OpenZeppelin, Base Sepolia, Reown AppKit, Wagmi, Viem, TanStack Query, SIWE-compatible messages, Anvil.

## Global Constraints

- Chain: Base Sepolia only.
- Web3 is a global deployment mode; points and token balances are never converted.
- Anonymous users may browse; wallet connection, signed login, confirmed deposit, and sufficient escrow balance are required to sit.
- Game actions and table settlement remain off-chain.
- Testnet contracts are not approved for mainnet or real-value assets.
